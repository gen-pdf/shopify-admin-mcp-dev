# Enum: `OrderRiskAssessmentCreateUserErrorCode`

Possible error codes that can be returned by `OrderRiskAssessmentCreateUserError`.

## Values

- `TOO_MANY_FACTS` — Too many facts were provided for the risk assessment.
- `ORDER_ALREADY_FULFILLED` — The order is marked as fulfilled and can no longer accept new risk assessments.
- `INVALID` — The input value is invalid.
- `NOT_FOUND` — The record with the ID used as the input value couldn't be found.
