# Enum: `CustomerPaymentMethodRemoteUserErrorCode`

Possible error codes that can be returned by `CustomerPaymentMethodRemoteUserError`.

## Values

- `INVALID` — The input value is invalid.
- `PRESENT` — The input value needs to be blank.
- `TAKEN` — The input value is already taken.
- `EXACTLY_ONE_REMOTE_REFERENCE_REQUIRED` — Exactly one remote reference is required.
- `AUTHORIZE_NET_NOT_ENABLED_FOR_SUBSCRIPTIONS` — Authorize.net is not enabled for subscriptions.
- `BRAINTREE_NOT_ENABLED_FOR_SUBSCRIPTIONS` — Braintree is not enabled for subscriptions.
